## ✨ Features

### 🗺️ Interactive Traffic Map (Leaflet)
- Real-time event markers on Bengaluru map
- Color-coded by event severity and type
- Click events to trigger full AI analysis
- Displays blocked roads (red), alternative routes (green), and normal roads
- Dynamic zoom to event location

### 🤖 AI Traffic Copilot (Left Sidebar)
- Natural language chat interface for operators
- Report incidents in plain English ("Vehicle breakdown on Hebbal Flyover")
- Auto-extracts structured event data and creates markers
- Powered by Google Gemini via backend API
- Tactical response formatting with action recommendations

### 🎯 Impact Dashboard (Bottom Left Panel)
- Displays predicted impact score (0-100) with visual gauge
- Shows impact level (Low/Medium/High/Critical)
- Model confidence indicator
- Top contributing factors with importance bars
- Real-time prediction on event selection

### 🚔 Resource Panel (Bottom Right Panel)
- Recommended officers, barricades, and tow vehicles
- Estimated resolution time
- Prediction confidence indicator
- Visual resource allocation cards

### 📊 Memory Explorer (Right Sidebar - Tab 1)
- Similar historical events retrieval
- Similarity scores with event details
- Average resolution time from similar incidents
- Historical success patterns
- Recommended actions based on precedent

### 🛣️ Diversion Simulator (Right Sidebar - Tab 2)
- Configure event location and impact radius
- Simulate road closures with one click
- View blocked roads, alternative routes, affected junctions
- Congestion risk score (0-100)
- Estimated extra travel time

### 📈 Performance Analytics
- Model performance metrics (MAE, RMSE, R², accuracy)
- Prediction count tracking
- Drift score monitoring
- Active model status

## 🧩 Component Architecture

```
src/
├── app/
│   ├── layout.tsx              # Root layout with header & fonts
│   ├── page.tsx                # Main dashboard page
│   └── globals.css             # Global styles & CSS variables
│
├── components/
│   ├── dashboard/              # Feature-specific components
│   │   ├── TrafficMap.tsx       # Leaflet map wrapper
│   │   ├── LeafletMap.tsx       # Full Leaflet implementation
│   │   ├── TrafficCopilot.tsx   # AI chat interface
│   │   ├── ImpactDashboard.tsx  # Impact scoring panel
│   │   ├── ResourcePanel.tsx    # Resource recommendation panel
│   │   ├── MemoryExplorer.tsx   # Similar events panel
│   │   ├── DiversionSimulator.tsx # Traffic simulation panel
│   │   └── PerformanceAnalytics.tsx # Model metrics panel
│   │
│   └── ui/                     # ShadCN UI primitives (35 components)
│       ├── button.tsx, card.tsx, dialog.tsx, tabs.tsx ...
│       └── (auto-generated by shadcn-ui CLI)
│
├── context/
│   └── DashboardContext.tsx     # Global state: events, predictions, loading
│
├── hooks/
│   ├── use-mobile.tsx          # Responsive breakpoint hook
│   └── use-toast.ts            # Toast notification hook
│
├── lib/
│   ├── api.ts                  # API client (fetch wrapper + TypeScript types)
│   ├── types.ts                # Shared type definitions
│   └── utils.ts                # Utility functions (cn helper)
│
└── ai/
    ├── genkit.ts               # Genkit AI initialization
    ├── dev.ts                  # Genkit dev server
    └── flows/                  # Genkit AI flow definitions
        ├── ai-traffic-copilot-query.ts
        ├── predictive-impact-analysis.ts
        ├── intelligent-resource-recommendation.ts
        └── diversion-path-simulation.ts
```

### Data Flow

```
User selects event on map
    → DashboardContext.selectEvent()
        → Parallel API calls to backend:
            • POST /predict-impact
            • POST /recommend-resources
            • POST /similar-events
            • POST /predict-resolution
        → Update state → Re-render panels
        → Background: POST /simulate-diversion
            → Update map with routes
```

## 🚀 Setup & Installation

### Prerequisites
- Node.js 20+
- npm 10+
- Running backend server (see `../backend/README.md`)

### 1. Install Dependencies

```bash
cd frontend
npm install
```

### 2. Configure Environment

```bash
cp .env.example .env.local
# Edit .env.local with your values
```

### 3. Start Development Server

```bash
npm run dev
```

The app will be available at [http://localhost:9002](http://localhost:9002)

### 4. Ensure Backend is Running

The frontend expects the backend API at `http://localhost:8000` by default. Start the backend:

```bash
cd ../backend
uvicorn app.main:app --reload --port 8000
```

## 🔗 Connecting to Backend

The frontend connects to the backend via the `NEXT_PUBLIC_API_URL` environment variable.

The API client is in `src/lib/api.ts`:

```typescript
const API_BASE_URL = process.env.NEXT_PUBLIC_API_URL || 'http://127.0.0.1:8000';
```

### Available API Methods

| Method | Endpoint | Description |
|---|---|---|
| `trinetraApi.getEvents()` | `GET /api/v1/events` | Fetch events from DB |
| `trinetraApi.predictImpact()` | `POST /api/v1/predict-impact` | Impact score prediction |
| `trinetraApi.recommendResources()` | `POST /api/v1/recommend-resources` | Resource recommendation |
| `trinetraApi.findSimilarEvents()` | `POST /api/v1/similar-events` | Similar events retrieval |
| `trinetraApi.predictResolution()` | `POST /api/v1/predict-resolution` | Resolution time prediction |
| `trinetraApi.simulateDiversion()` | `POST /api/v1/simulate-diversion` | Diversion simulation |
| `trinetraApi.getModelPerformance()` | `GET /api/v1/model-performance` | Model metrics |
| `trinetraApi.submitFeedback()` | `POST /api/v1/feedback` | Prediction feedback |
| `trinetraApi.copilotQuery()` | `POST /api/v1/copilot` | AI Copilot query |

## 🐳 Docker

### Build & Run (Standalone)

```bash
cd frontend
docker build \
  --build-arg NEXT_PUBLIC_API_URL=http://your-backend-url:8000 \
  -t trinetra-frontend .
docker run -p 3000:3000 trinetra-frontend
```

### With Docker Compose (Recommended)

See the root-level `docker-compose.yml` for full-stack orchestration.

```bash
# From project root
docker-compose up --build
```

The frontend will be available at [http://localhost:3000](http://localhost:3000)

## 🎨 Design System

The UI uses a custom dark theme with CSS variables:

- **Background**: Deep dark (`#020408`)
- **Primary**: Blue accent (HSL-based)
- **Fonts**: Inter (body), Space Grotesk (headlines)
- **Glassmorphism**: `backdrop-blur-xl` with semi-transparent borders
- **Animations**: Smooth transitions, pulse indicators
- **Layout**: Resizable split panels with drag handles
